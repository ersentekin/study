c functions

###C-functions

simple example :

void myName (NSString *name)
{
    NSLog(@"hello : %@", name);
}

if you are NOT inside @implementation block, compiler will complain about that could not see its declaration with following compiler error:

implicit declaration of function X is invalid in C99

so you should put complete definition above your function call line OR declare it above call line and implement it below call line.

  • If you put static keyword like this:
static void myName (NSString *name)
{
    NSLog(@"hello : %@", name);
}

it will limit the scope of the function to the current file. If NOT static, then it will be global WITHOUT importing the file of the function, all over the project.

  • if you make a local variable as static :
static void myName (NSString *name)
{
	static int counter = 0;
	counter++;
    NSLog(@"hello : %@ and count : %d", name, counter);
}

it will REMEMBER its value when you call function, then count like 1, 2, ,3…

  • inline functions:
static inline void myName (NSString *name)
{
    NSLog(@"hello : %@", name);
}

inline says to compiler that, put this function code block to the related function called line (inline ;)) and thereby saving the overhead of a function call.

  • NS_INLINE macro means __attribute__((always_inline)) compiler directive. stronger than normal inline.
